Build a complete desktop-based AI Agent Operating System called AgentSpace that enables developers and teams to manage software projects through multiple collaborating AI agents working in parallel. 
The system should provide a unified workspace where users can create or import projects, organize work through task boards, launch and monitor multiple AI agents, manage code repositories, and coordinate development activities from a single interface. 
Users should be able to assign tasks to specialized agents such as architecture, frontend, backend, database, testing, review, documentation, and research agents, while a central orchestration engine automatically distributes work, tracks progress, resolves conflicts, and combines results. 
The platform should include a multi-terminal environment where each terminal can run an independent AI agent session, allowing several development tasks to execute simultaneously. A visual task management system should support planning, prioritization, dependency tracking, review workflows, and automatic status updates generated by agent activity. 
The application must maintain a persistent project memory containing requirements, architecture decisions, coding standards, implementation notes, previous conversations, task history, and agent-generated knowledge so that all agents share a consistent understanding of the project. 
The system should automatically analyze repositories to understand project structure, dependencies, modules, technologies, and architecture, then make this information available to both users and agents. Include an integrated file explorer, code editor, project dashboard, notification center, command history viewer, activity timeline, session replay capability, and agent communication layer.
Agents should be able to collaborate through shared memory, exchange findings, request assistance from other agents, and coordinate implementation efforts while respecting file ownership rules that prevent editing conflicts. Support voice-based project management where spoken instructions can be converted into tasks and assigned to appropriate agents. Provide repository management features including branch awareness, change tracking, commit history visualization, code review workflows, and project progress monitoring. 
The interface should present a modern productivity-focused workspace with project navigation, agent management, task boards, editor views, terminal panels, memory exploration, notifications, analytics, and real-time agent status monitoring. The architecture must be modular, scalable, extensible, secure, and designed to support future capabilities such as cloud synchronization, team collaboration, remote agents, plugin ecosystems, agent marketplaces, advanced automation workflows, and enterprise deployments. 

Before implementation, generate a complete product architecture, user flows, system design, database design, API specifications, folder structure, permission model, agent workflow definitions, UI wireframes, and a phased development roadmap, then proceed with building the full production-ready application.
